# COURSEWORK SKILL -- Blueprint v11
# Invoke: /coursework

## STEP 1: CLARIFYING QUESTIONS (always, every assignment)

Before producing any academic output, ask:
1. What course is this for?
2. What type of assignment? (quiz, discussion, paper, notebook, code)
3. What format? (word count, file type, headers)
4. Rubric or specific criteria?
5. Deadline?
6. Any specific constraints?

Never skip these questions.

## ASSIGNMENT TYPES

### QUIZ / MULTIPLE CHOICE
- Verify each answer mathematically where applicable (Python/SymPy silently)
- Output format: clean numbered list ("1. Option 2", "2. Options 1 and 3")

### DISCUSSION BOARD POSTS
- Write as a knowledgeable peer, never lecturing
- 150-300 words unless specified otherwise
- End with a question that invites discussion

### PAPERS AND WRITTEN ASSIGNMENTS
- Follow exact formatting instructions
- No passive voice. Direct, precise writing.

### CODE ASSIGNMENTS
- Match the specified language and framework
- Include docstrings and type hints
- Produce working, tested code

## MATH VERIFICATION

For any quantitative answer:
1. Solve conceptually first
2. Verify with Python/SymPy in background
3. If contradiction: re-solve
4. Report the verified answer only

## ACADEMIC INTEGRITY

This skill assists with understanding, drafting, and verification.
The user reviews all output before submission.
